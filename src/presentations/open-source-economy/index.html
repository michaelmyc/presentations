Source Code vs. Binary

  • Source cod and binary are two important constructs of a piece of software
  • Source code is the code humans read and write in order to build software, consider it as drawings by an architect
  • Binary is a set of difficult-to-understand instructions intended for a machine to execute, consider it as orders given at a construction site
  • The software you install on your computer are binaries, and you will need access to source code to make modifications to a piece of software

Open source software is software whose source code is open to the public.

A Brief History of Open Source

  • Before going into the open source economy of today, let's briefly go over how open source was created.

Richard M. Stallman

  • The story of OSS starts with RMS
  • Programmer @ MIT in late 1970s
  • Computers and printers were large and expensive, thus shared usage
  • They're also unreliable (lots of jamming)
  • MIT folks devised of a system to send a message to everyone who has a job in queue that the printer is jammed.
  • However to do this, needs to modify printer driver
  • 1980, the lab got new laser printer via donation
  • RMS was refused when he asked company for source code of the driver
  • RMS felt closed-source proprietary software betrayed hacker culture

GNU Not Unix

  • 1983, RMS announced the GNU project, which:
    • Aims to be an open source OS compatible with the popular proprietary Unix
    • Stands for "GNU Not Unix"
    • Most software written under GNU, but uses the Linux kernel, which is written by Linus Torvalds. Story for another day.
    • ~80% of servers run on Linux (source Wikipedia)
    • 100% of top 500 supercomputers run on Linux (source Wikipedia)

MongoDB

State-of-the-art NoSQL database

Business Model

Free DB software

Paid multi-cloud hosted database (Atlas)

Atlas reduces operational costs for customers

Paid customers prioritized in support and maintenance requests

  • Deployment of a cloud-based DB system and managing backups and server downtimes is highly technically demanding and costly for businesses
  • Prevalence of MongoDB as a free solution makes developers more familiar with it
  • These developers may one day make decisions on what db to use in their companies, making them potential customers
  • Essentially, open source marketing

Red Hat Enterprise Linux (RHEL)

Enterprise Linux distribution

Business Model

Free download

Tiered subscriptions of solutions

Handles support and maintenance

Reduces procurement costs for customers

  • Tiered subscription of solutions targeting different business scenarios
  • RHEL promises that the packages included are tested and will have support and maintenance from RedHat
  • Companies using RHEL procures one product, rather than procuring each OS package they may need to use (procurement costs upwards of 20k per iteration for big companies)
  • Sells the promise of support for OS software
  • Extremely profitable, with more than $17 billion revenue and ~85% gross profit margins

Chromium (Google)

Open source browser engine

Benefits for Google

Increased security

Reduced maintenance and testing costs

Free customer research

Market dominance == power over Web standards

  • More eyes auditing the code == less likely to have unknown vulnerabilities being exploited
  • Free beta testing == less testers needed in house
  • Community patches == less maintenance costs
  • Developers (customers of a browser engine) tell you what features are desired, essentially free market research
  • Chromium-based browser has ~70% market share in all browsers and ~80% in desktop browsers (source Wikipedia)
  • Power over Web standards will allow Google to prevent push of privacy features that hurt their advertising business

React JS (Meta)

Front-end development framework

Benefits for Meta

Doesn't generate revenue if not open sourced

Reduced maintenance and testing costs

Invisible benefits including reduced training costs, better company image, etc.

  • Dev framework does not generate revenue even if closed sourced
  • New hires most likely have exposure to popular Open Source tool, thus reducing training costs
  • Open sourcing useful dev tools creates better company image (open source friendly and engineering excellency) and likely helps Meta hire better engineers
